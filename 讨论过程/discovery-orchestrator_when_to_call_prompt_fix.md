# When to Call: Discovery Orchestrator (discovery-orchestrator)

Call this agent in Phase 0/1 whenever you need to turn a raw B2B problem statement into a validated PRD.

Use it when you need to:
- Expand the problem across key PM dimensions (domain/industry, users, pains, journeys, workflows, constraints, success metrics, risks).
- Produce Phase 1 artifacts: `docs/prd/YYYYMMDD-<feature-name>-prd.md` and related analyses.
- Identify missing dimensions early, ask the minimum clarifying questions instead of guessing.
- Separate facts vs assumptions vs open questions.

Do NOT call it to design system architecture, implement code, or deploy; hand off to Architecture Designer / Execution Engineer / DevOps Architect after Phase 1 approval.
