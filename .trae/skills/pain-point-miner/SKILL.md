# 痛点与干系人分析师 (Pain Point Miner)

本 Skill 是 B 端产品发现的灵魂，负责将模糊的原始需求拆解为结构化的干系人诉求与根因，直接为 C0 的 `stakeholder_map` 与 `critical_scenarios` 供弹。

## 1. 核心职责
- **干系人映射**：穷举并区分业务决策者（Buyer）、管理者（Supervisor）与执行者（End User），梳理其权责利冲突。
- **AS-IS 旅程还原**：模拟目标用户在没有新系统前的真实工作流，找出信息断点、重复劳动与高风险环节。
- **5Why 根因追溯**：剥离表层症状（如“审批慢”），找到系统性根因（如“数据孤岛导致需要反复核对线下表格”）。

## 2. 依赖工具与 MCP
- 强依赖：逻辑推理与 B 端经验模型
- 建议 MCP（针对有内部数据的场景）：`pdf-parser` (分析历史访谈) / `sql-tool` (分析日志)

## 3. 使用方式
```text
指令：使用 pain-point-miner 拆解 [原始问题] 的干系人与根因。
输入：一段原始业务痛点描述
输出预期：
- 干系人地图（Role -> 核心诉求 -> 现状痛点）
- AS-IS 关键流转路径与断点
- 经过 5Why 提炼后的真实问题定义
```
