# PRD｜配电房巡检缺陷检测与安全作业卡（能源）

- 文档版本：v0.1（Phase 1 草案）
- 日期：2026-03-26
- 负责人：Discovery（Phase 0/1 PM）
- 适用对象：电网公司配电房巡检与缺陷治理闭环

---

## 1. 背景与问题陈述

### 1.1 业务背景（事实）
- 场景：电网公司对配电房开展周期性巡检。
- 数据：现场采集多源影像，包含可见光与红外（照片/视频）。

### 1.2 主要痛点（事实）
- 人工识别容易漏检、误判，质量难一致。
- 作业票据合规风险高（票据不规范、要素缺失、审批链不清）。
- 现场回传不规范（素材命名/关联设备/时间地点/过程留痕不一致）。

### 1.3 业务目标（事实）
实现“缺陷检测 → 风险分级 → 自动作业卡 → 闭环复核”的端到端流程，并满足合规留痕与审计要求。

---

## 2. 目标、范围与成功标准

### 2.1 产品目标（事实）
- 提升巡检缺陷发现率、降低漏检率。
- 将检测结果标准化落地到风险分级与作业卡生成，减少人工票据编制成本与合规风险。
- 建立闭环复核机制：从发现到整改到复核闭环可追溯。

### 2.2 非目标（明确边界）
- 不在本期直接替代/重构企业现有的全量资产管理系统（如已有 EAM/OMS/工单系统），仅做必要集成与数据同步。（假设：企业存在或可引入对接点）
- 不承诺在无历史标注数据情况下立即达到特定检测精度指标（精度目标需基于试点数据评估后冻结）。（假设）

### 2.3 成功指标（需在试点中量化确认）
- 过程指标：
  - 巡检回传合规率（关键字段齐全、素材可追溯）≥ X%（待定）
  - 自动生成作业卡采用率 ≥ X%（待定）
  - 缺陷初判到工单/作业卡生成时长降低 ≥ X%（待定）
- 结果指标：
  - 漏检率降低 ≥ X%（待定，需定义“漏检”的口径与复核方法）
  - 重大风险缺陷发现时效提升 ≥ X%（待定）

> 说明：以上 X% 为**待定指标**，需要结合试点规模、缺陷基准集与现网流程确认，避免在无数据支撑下编造目标值。

## 3. 目标用户与角色

### 3.1 角色列表（事实 + 合理补全）
- 巡检员（现场）：采集照片/视频（可见光/红外）、补充现场信息、提交巡检记录。
- 班组长/巡检主管：复核巡检质量、确认缺陷、分派整改。
- 安全员/安监：审核作业卡合规性（安全措施、风险等级、人员资质、审批链）。
- 缺陷/设备专责（运维工程师）：确认缺陷类型与处置方案，形成整改闭环。
- 审计/合规（内控）：抽查留痕、导出审计证据链。
- 系统管理员：配置设备台账映射、作业规程分级、角色权限、模型/规则版本。

### 3.2 关键使用场景
- 现场巡检：弱网/忙碌环境下快速采集、自动校验并标准化回传。
- 缺陷审核：集中看板批量处理，提高复核效率。
- 生成票据：一键生成作业卡并走审批流，保证要素齐全与可追溯。
- 闭环复核：整改前后对比、证据链完整、可审计。

## 4. 核心流程（To-Be）

### 4.1 端到端流程（目标态）
1) 巡检采集：现场采集可见光/红外照片或视频，系统引导规范拍摄与命名（设备/位置/工位）。
2) 回传规范化：自动提取时间/地点/设备关联信息；对素材质量做校验（清晰度、是否缺少关键视角）。
3) 缺陷检测：对多源影像执行缺陷识别，输出候选缺陷（位置/类型/置信度/截图或热像特征）。
3a) 历史对比变化检测（Change Detection）：使用 ChangeFormer 架构的变化检测技术，对比历史图像（如上次巡检 vs 本次巡检），识别“异常变化”，以降低漏检并提升风险分级可靠性。  
   - 输入：同一设备、同一视角/工位的历史基线影像（baseline）与本次巡检影像（可见光或红外，优先同模态对比），以及必要的设备/视角标识（用于检索与配对）。  
   - 输出：变化区域（mask/框/热力图）、变化置信度/变化评分、关联的“历史-当前”素材对、可追溯的模型版本；变化结果作为风险分级与复核的补充证据（例如：变化显著但单次检测置信度低时触发“复核关注”）。  
   - 边界/前置：需要“可用历史影像基线”且可对齐到同设备同视角（必要时做图像配准/对齐）；若缺少基线或无法可靠对齐，则跳过变化检测并显式标记原因（baseline 缺失/视角不一致/对齐失败），避免误导分级。
4) 风险分级：结合设备台账、作业规程分级与缺陷类型/严重度，生成风险等级与处置建议。
5) 自动作业卡：按风险等级与规程模板生成作业卡（作业内容、安全措施、人员/资质要求、工器具、停送电/隔离步骤、审批链等）。
6) 人工复核与审批：主管/专责复核缺陷与风险等级，安监审核票据合规，发起或同步到工单系统。
7) 执行与过程留痕：现场执行按步骤打卡（关键节点拍照/视频/测温），异常可上报。
8) 闭环复核：整改后复核影像与记录，确认缺陷消除/风险解除，归档可审计证据链。

### 4.2 关键状态机（巡检缺陷）
- 已采集 → 已回传（待检测）→ 已检测（待复核）→ 已确认（待分级/待票据）→ 已生成作业卡（待审批）→ 执行中 → 待复核 → 已闭环 / 已驳回（回到待复核/待补充）

## 5. 功能需求（含用户故事与验收条件）

> 说明：验收条件使用 Given/When/Then；所有涉及合规与审计的动作都必须生成不可篡改的操作留痕（见第 6 章）。

### 5.1 巡检采集与回传规范化

**FR-1 采集任务与设备定位**
- 用户故事：作为巡检员，我需要看到今天的巡检任务清单与配电房/设备范围，以便按计划完成巡检并避免漏项。
- 验收条件：
  - Given 巡检员已登录且具备该站点权限  
    When 打开“今日任务”  
    Then 展示站点/房间/设备清单、必拍视角要求、计划时间窗口与完成状态

**FR-2 多源影像采集（可见光/红外）**
- 用户故事：作为巡检员，我需要同时采集可见光与红外影像并关联到同一设备/位置，以便后续进行缺陷检测与对比分析。
- 验收条件：
  - Given 巡检员选择某设备  
    When 分别采集可见光与红外照片/视频并提交  
    Then 系统将两类素材绑定到同一设备与巡检记录，并记录采集时间、采集人、终端信息

**FR-3 素材质量校验与缺口提示**
- 用户故事：作为巡检员，我希望系统自动提醒素材是否模糊/曝光异常/缺少关键视角，减少回传后返工。
- 验收条件：
  - Given 巡检员提交素材  
    When 系统检测到清晰度不足或缺少规定视角  
    Then 以可操作的提示告知原因与补拍建议，并允许“补拍后提交”或“备注说明后提交（需复核关注）”

**FR-4 现场回传规范化（元数据补全）**
- 用户故事：作为主管，我希望每条巡检记录都能自动带齐设备台账映射、时间地点、任务编号等关键字段，便于审计与追溯。
- 验收条件：
  - Given 设备台账已导入且存在唯一设备标识  
    When 巡检员提交巡检记录  
    Then 系统自动写入设备ID、站点/房间、采集时间、操作者、任务ID、素材哈希（用于防篡改校验）

### 5.2 缺陷检测（多源影像）

**FR-5 缺陷候选生成（检测）**
- 用户故事：作为缺陷专责，我希望系统从可见光/红外影像中自动识别缺陷候选并给出定位与置信度，减少人工逐帧查看。
- 验收条件：
  - Given 某设备的巡检素材已回传完成  
    When 系统触发检测任务  
    Then 输出缺陷候选列表（缺陷类型、位置标注/截图、置信度、关联素材、检测模型版本）

**FR-5a 历史对比变化检测（Change Detection，ChangeFormer）**
- 用户故事：作为复核人员/缺陷专责，我希望系统能自动对比同设备同视角的历史影像与本次影像，识别异常变化并沉淀为可解释证据，以降低漏检并提升风险分级可靠性。
- 验收条件：
  - Given 该设备存在可用的历史基线影像且与本次影像可匹配到同一视角/工位  
    When 系统触发“变化检测”任务（基于 ChangeFormer 架构）  
    Then 输出变化区域（mask/热力图或框）、变化评分/置信度、关联的历史-当前素材对与模型版本，并可在缺陷详情与风险分级依据中被引用
  - Given 该设备缺少可用历史基线影像或同视角匹配失败  
    When 系统尝试触发“变化检测”  
    Then 系统不输出变化结论并明确标记原因（baseline 缺失/视角不一致/对齐失败），且不将该步骤结果用于自动升降级

**FR-6 多源一致性证据呈现（可见光/红外联动）**
- 用户故事：作为复核人员，我希望能在同一界面联动查看可见光与红外证据（同一位置的对照），提升确认效率。
- 验收条件：
  - Given 某缺陷候选同时关联可见光与红外素材  
    When 复核人员打开缺陷详情  
    Then 可一键切换/并排展示两类素材与同位置标注，并保留查看轨迹

**FR-7 人工纠正与反馈闭环（标注/确认）**
- 用户故事：作为专责，我需要对模型误检/漏检进行纠正（确认/驳回/补充缺陷），并沉淀为后续优化的数据。
- 验收条件：
  - Given 复核人员具备“缺陷复核”权限  
    When 对缺陷候选执行“确认/驳回/改类型/改等级/补充缺陷”  
    Then 系统记录变更前后内容、操作者、时间、原因，并生成可用于训练的数据条目（含版本与来源）

### 5.3 风险分级（规程驱动）

**FR-8 规则/规程驱动的风险分级**
- 用户故事：作为安监，我希望风险分级遵循企业作业规程分级规则且可追溯到规则版本，避免“拍脑袋分级”。
- 验收条件：
  - Given 缺陷已被确认且设备台账与规程规则可用  
    When 系统执行风险分级  
    Then 输出风险等级、分级依据（规则条款引用/版本）、建议处置等级与建议时限

**FR-9 分级可配置与审批生效**
- 用户故事：作为系统管理员，我需要在后台维护分级规则与模板，并通过审批后生效，确保规则变更受控。
- 验收条件：
  - Given 管理员提交规则变更  
    When 规则进入“待审批”并被授权人批准  
    Then 新规则以新版本生效，历史记录仍使用旧版本并可追溯

### 5.4 自动作业卡（安全作业票据）

**FR-10 作业卡模板库（按风险与作业类型）**
- 用户故事：作为安监，我需要维护不同风险等级/作业类型的作业卡模板（必填项、措施清单、审批链），确保一致合规。
- 验收条件：
  - Given 管理员进入模板库  
    When 新建/编辑模板并配置必填项与审批链  
    Then 模板保存为版本化实体，发布后可被生成引擎引用

**FR-11 一键生成作业卡（从缺陷到票据）**
- 用户故事：作为专责/主管，我希望从“已确认缺陷”一键生成作业卡，系统自动填充关键信息并提示缺失项。
- 验收条件：
  - Given 缺陷已确认且已完成风险分级  
    When 点击“生成作业卡”  
    Then 自动生成作业卡草稿（设备信息、缺陷描述、风险等级、推荐措施、需上传证据清单），并标记未满足的必填项

**FR-12 电子签署与审批流（合规）**
- 用户故事：作为安监/主管，我需要按权限完成电子签署与审批流转，形成完整可审计链路。
- 验收条件：
  - Given 作业卡草稿已补齐必填项  
    When 发起审批  
    Then 系统按预设审批链流转并记录每一步签署人、时间、意见、签署方式（电子签/账号确认等，待定）

### 5.5 执行留痕与闭环复核

**FR-13 执行步骤打卡与证据采集**
- 用户故事：作为现场执行人员，我需要按作业卡步骤逐项打卡，并上传关键证据，避免事后补录不规范。
- 验收条件：
  - Given 作业卡已审批通过且进入“执行中”  
    When 执行人员完成某步骤并提交证据  
    Then 系统记录步骤完成时间、执行人、证据素材、地点（如可用）、并禁止跳过必需步骤（除非授权人豁免并留痕）

**FR-14 闭环复核与归档**
- 用户故事：作为主管/专责，我需要对整改结果进行复核并最终关闭缺陷，形成可审计归档包。
- 验收条件：
  - Given 作业执行完成并提交复核材料  
    When 复核人点击“通过/驳回”  
    Then 缺陷状态更新为“已闭环/待整改”，并生成归档包索引（含所有关键证据的引用与哈希）

**FR-15 审计查询与导出**
- 用户故事：作为审计人员，我需要按时间/站点/设备/作业卡编号查询全过程记录并导出证据链。
- 验收条件：
  - Given 审计人员具备只读审计权限  
    When 选择条件并导出  
    Then 导出包含：巡检记录、检测与复核结果、规则/模型版本、作业卡与审批链、执行留痕、闭环结论、素材清单与哈希

## 6. 非功能要求（合规留痕 / 审计重点）

### 6.1 合规留痕（必须）
- 全链路留痕：采集、上传、检测、复核、分级、生成作业卡、审批、执行、复核关闭的每一步都要记录操作者、时间、动作、对象、前后差异。
- 防篡改：对关键证据（图片/视频/作业卡 PDF/结构化记录）生成哈希；对记录追加式写入（Append-only）。（实现方式在 Phase 2 决定）
- 版本可追溯：检测模型版本、分级规则版本、作业卡模板版本必须写入业务记录并可审计回放。
- 证据保全：所有素材与关键表单需具备保留策略（期限、访问控制、导出水印/脱敏策略待定）。

### 6.2 安全与权限
- 最小权限：按角色控制查看/编辑/审批/导出权限；支持站点/班组级的数据隔离。
- 数据安全：传输加密（TLS）、存储加密（如有要求）、敏感字段脱敏展示（审计除外）。
- 账号与身份：对接企业统一身份认证（SSO/LDAP 等）。（假设：企业有统一身份体系）

### 6.3 性能与可用性（待试点确认指标）
- 检测时延：单设备素材从回传完成到生成缺陷候选 ≤ X 分钟（待定，取决于算力与并发）。
- 并发：支持同时回传/检测的巡检点数量 ≥ X（待定）。
- 可用性：核心服务月可用性 ≥ 99.5%（待定）。

### 6.4 可运维性
- 可观测：检测任务、回传失败、审批超时、闭环超期等关键事件告警。
- 可追责：任何关键字段修改必须带原因与审批（如需）。

## 7. 数据与集成需求（高层）

### 7.1 输入（事实）
- 现场照片/视频（可见光/红外）。
- 设备台账（设备ID、位置、型号、投运信息等）。
- 作业规程分级规则（风险等级规则、作业卡模板、审批链规则）。
- 合规留痕要求（审计字段、保留期限等）。

### 7.2 输出
- 缺陷候选与复核结论（结构化）。
- 风险分级结果与依据。
- 自动生成的作业卡（结构化 + 可归档格式）。
- 闭环归档包索引与审计导出。

### 7.3 外部系统对接（开放项）
- 资产/设备台账系统（主数据同步）。
- 工单/缺陷管理系统（生成工单或同步状态）。
- 统一身份认证系统（SSO）。
- 电子签章/CA（如需强合规签署）。

> 注：具体对接清单、接口协议、主从关系在 Phase 2 设计冻结。

## 8. 风险、假设与开放问题（明确标注）

### 8.1 关键风险
- 数据风险：历史缺陷样本与标注不足导致模型效果不可控；历史影像基线可用性不足（缺失/保留期限短/不可检索）会削弱历史对比变化检测；同设备同视角对齐（视角漂移、焦距变化、遮挡）与图像配准失败会引入误报/漏报；红外与可见光跨模态对齐难，影响多源证据一致性与跨模态变化解释。
- 合规风险：电子签署、留痕防篡改、保留期限等要求不清会导致方案返工。
- 流程风险：现网已有工单/票据系统，若集成边界不清会出现“双轨运行”与责任不清。
- 现场风险：弱网/断网场景导致回传与打卡中断，影响闭环完整性。

### 8.2 假设（需要被验证）
- 假设 A1：设备台账具备可用的唯一设备标识，并能映射到巡检任务与现场采集。
- 假设 A2：企业允许将巡检影像与票据数据集中存储（或提供专网/私有化部署条件）。
- 假设 A3：作业规程分级规则与作业卡模板可以结构化（条款/要素可配置），且有明确维护责任人。

### 8.3 开放问题（最多 5 个；缺一会影响方案冻结）
1) 交付形态：现场端是既有 App/手持终端还是需要新建移动端？是否必须支持离线采集与断点续传？
2) 缺陷体系与数据：当前是否已有标准缺陷字典/严重度定义/典型样例库（可见光+红外）？历史巡检影像是否可检索且满足保留期限（用于历史对比变化检测）？是否能保证同设备同视角采集规范/对齐条件（含红外/可见光跨模态对齐诉求）？
3) 风险分级：分级规则由哪个部门维护？是否需要“规则变更审批”与生效窗口？依据条款如何引用到票据中？
4) 票据合规：作业卡是否必须接入电子签章/CA？审计保留期限与不可篡改要求的具体口径是什么？
5) 系统集成：是否需要对接现有工单/缺陷管理系统？主系统是谁、期望的最小集成字段与状态回写有哪些？

## 9. 里程碑（建议）
- M0（2-4 周）：需求确认 + 试点数据盘点 + 缺陷字典/规则/模板梳理（形成可配置原型）。
- M1（4-8 周）：回传规范化 + 缺陷检测 MVP + 复核台（闭环留痕打通）。
- M2（8-12 周）：风险分级 + 自动作业卡 + 审批流 + 审计导出（可试点上线）。

> 以上为建议节奏，最终以资源与试点范围为准。

## 10. 附录：名词解释（可选）
- 缺陷：影响设备安全/可靠运行的异常现象或隐患（具体口径以企业标准为准）。
- 作业卡：安全作业票据的标准化载体，包含风险措施、审批链与过程留痕要求。
