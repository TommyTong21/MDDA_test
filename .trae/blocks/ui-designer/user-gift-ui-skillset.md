# UI Designer 增强：如何使用 user_gift（离线技能簇）

## 何时调用

- 需要提升 UI 设计产出质量（信息架构、交互模式、表单/表格/弹窗范式）
- 需要从成熟组件生态中抽取“可落地的设计-实现规范”沉淀为积木

## 核心原则

- `user_gift/` 存放的是完整工程/技能簇，只作为离线参考
- 主工程只沉淀“轻量可复用积木”（`.trae/blocks/`），避免把大仓库直接引入业务工程

## 四套资源的定位

- `user_gift/shadcn-ui-main/`：Web 端组件与设计范式参考（按需生成/拷贝）
- `user_gift/refine-main/`：Admin（CRUD/权限/资源化路由）方法论参考
- `user_gift/gluestack-ui-main/`：移动端（Expo/RN）组件与模式参考
- `user_gift/arco-design-skill-main/`：UI 知识增强（组件 API、模式指南），适合作为“设计-实现一致性”参考

## 输出要求（沉淀到积木库）

每次参考 user_gift 必须产出至少一条积木（Markdown），并满足：
- 适用范围：Web / Admin / App
- 落盘路径：明确写出 `.trae/blocks/...`
- 可执行步骤：pnpm 命令/配置片段/文件模板
- 验证方式：最小页面 + E2E 冒烟或截图验收点

## 推荐沉淀主题清单（优先级从高到低）

- 表单：字段联动、校验、提交态、错误态、可访问性
- 表格：筛选/排序/分页/批量操作/行内编辑
- 弹窗：确认/表单弹窗/抽屉/二次确认
- 权限：按钮级/页面级可见性与错误态
- 空状态：无数据/无权限/加载中/失败重试
